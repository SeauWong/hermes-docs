GEPA — 反思式提示进化

自然语言反思超越策略梯度。比 GRPO 少用 35 倍的 rollout。超越 MIPROv2 13%。原生支持闭源模型。

核心洞察

语言是比标量奖励更丰富的学习媒介。一次 rollout 产生的自然语言轨迹——推理链、工具输出、编译器报错——已经包含了足够的诊断信号，让 LLM 能有效修改自己的提示词。

以 GRPO 为代表的强化学习方法需要数万次 rollout，因为它们只能从每次 rollout 结束时的稀疏标量奖励中学习。GEPA 则把每一次执行轨迹当作可解释的学习信号——就像人类工程师调试系统时做的那样。一次 rollout 的反馈就能产生巨大的行为更新。

三大支柱

（Re）反思

LLM 读取执行轨迹、工具输出和评估反馈（例如"编译器错误：第 42 行类型不匹配"），诊断问题所在，并提出修改后的提示词。提示词的更新是自然语言——可解释、可组合、高信号。

（Ge）遗传

每个新候选都从父代继承积累下来的"经验教训"，形成一棵进化树。不同于一次性提示优化，GEPA 跨代积累知识——子代提示携带祖先发现的策略。

（Pa）Pareto 采样

不总是变异最优候选（那样很快会停滞）。GEPA 为每个训练实例维护最佳候选，构成 Pareto 前沿，然后随机采样——平衡探索新策略和利用已验证策略。

范式转换

强化学习（GRPO）

  • × 只从标量奖励学习
  • × 每个任务需要 24,000+ rollout
  • × 需要权重更新（LoRA 或全量微调）
  • × 无法优化闭源模型
  • × 聚合提升：+3.68%（Qwen3 8B）

GEPA（自然语言反思）

  • ✓ 从完整执行轨迹学习
  • ✓ 402–7,051 rollout 达到最优（最多少 78 倍）
  • ✓ 仅提示词优化——无需权重更新
  • ✓ 支持 GPT-4.1 Mini、Gemini 等任何 API 模型
  • ✓ 聚合提升：+9.62%（Qwen3 8B）

系统架构

GEPA 接收一个复合 AI 系统、训练集、反馈函数和 rollout 预算，然后迭代地提出更好的候选。

反思式变异

LLM 反思轨迹，提出提示词更新，在 minibatch 上测试

反馈函数

GEPA 将标准评估指标扩展为反馈函数，同时返回数值评分和自然语言轨迹——编译器错误、评分细则失败、性能分析数据。这些轨迹是反思之所以可能的诊断信号。

核心算法

公式 1：逐实例 Pareto 前沿

Pareto 前沿构建
$$s^*[i] = \max_{k} S_{\mathcal{P}[k]}[i]$$

对每个训练实例 i，记录候选池中所有候选的最高得分。这定义了 Pareto 前沿——迄今为止发现的"获胜"策略集合。

公式 2：选择概率

加权随机采样
$$P(\Phi_k) \propto f[\Phi_k] = |\{i : \Phi_k \in \hat{\mathcal{P}}^*[i]\}|$$

候选按"赢的次数"加权采样——在 5 个实例上领先的候选被选中的概率是在 1 个实例上领先的候选的 5 倍——但"窄领域专家"仍会被探索。

公式 3：优化循环

完整 GEPA 算法
$$\Phi' = \mathrm{Copy}(\Phi_k) \text{ with module } j \text{ updated by } \pi_j' \leftarrow \mathrm{Reflect}(\text{traces, feedback})$$

LLM 反思性地归因成功或失败到提示词元素，提出新指令，然后在同一 minibatch 上测试。只有改进后的候选才加入候选池。

优化流程

  1. 初始化：候选池 \(\mathcal{P} = [\Phi]\) 包含基础系统。将训练数据分为反馈集和 Pareto 集。
  2. Pareto 选择：构建逐实例 Pareto 前沿。按"赢的次数"加权随机采样一个候选进行变异。
  3. 模块选择：round-robin 选择要优化的模块（确保所有模块都得到更新）。
  4. Rollout 与反思：在 minibatch 上运行候选。收集执行轨迹、工具输出和评估反馈。LLM 反思这些并提出新提示词。
  5. Minibatch 测试：如果新提示词在 minibatch 上改进了，加入候选池并记录祖先关系，然后在完整 Pareto 集上评估。
  6. 重复直到预算 \(B\) 耗尽。返回 Pareto 集上聚合得分最高的候选。

实验设置

6 个基准

HotpotQA（多跳问答）、IFBench（指令遵循，OOD 约束）、HoVer（检索增强验证）、PUPA（隐私感知委托）、AIME-2025（数学）、LiveBench-Math（泛化能力）。

2 个模型，4 个优化器

Qwen3 8B（开源）和 GPT-4.1 Mini（闭源）。对比 GRPO（24k rollout + LoRA）、MIPROv2、TextGrad 和 Trace（OptoPrime）。

实战走一遍：对抗性提示搜索

GEPA 的奖励信号可以反转——不是最大化性能，而是最小化它。结果：一个对所有查询都有效的通用对抗指令。

  1. 设置：从 AIME 数学题的干净指令提示开始。目标：GPT-5 Mini，AIME-2025（30 题，每题 5 次 = 150 次生成）。
  2. 进化：GEPA 迭代地提出添加信息（如冷知识）的提示编辑，同时保持任务可解。目标：降低 pass@1。
  3. 结果：一条对抗指令将 pass@1 从 76% 降到 10%——错误率提升 3.8 倍。注入的干扰信息（蜂蜜不腐、尼罗河最长、海豚睁眼睡觉）导致模型系统性误解格式规则。
  4. 为什么重要：这说明 GEPA 的反思能力足以自动发现指令遵循的脆弱交互——产生可复用的压力测试，用于模型安全评估。

边界与限制

何时会停滞

如果 Pareto 前沿收敛到单一策略且所有变异都无法改进，GEPA 会耗尽预算却找不到更好的提示。这发生在搜索空间太窄或反馈函数提供弱诊断信号时。

Merge 时机

GEPA+Merge 提升 GPT-4.1 Mini 但在 Qwen3 8B 上某些任务反而下降。交叉操作（Merge）对两个模型用了相同的超参数，对 Qwen 来说不是最优。Merge 只在足够不同的进化分支出现后才最有效。

验证集预算

GEPA 的大部分 rollout 预算花在 Pareto（验证）集上评估候选用于选择，而非用于学习。进一步用更小的验证集或动态子集可以获得更高的效率。
