GEPA — REFLECTIVE PROMPT EVOLUTION

Natural language reflection beats policy gradients. Up to 35x fewer rollouts than GRPO. +13% over MIPROv2. Works on closed-source models out of the box.

The Core Insight

Language is a richer learning medium than scalar rewards. A single rollout's natural language trace — reasoning chains, tool outputs, compiler errors — gives an LLM enough diagnostic signal to mutate its own prompts effectively.

RL methods like GRPO require tens of thousands of rollouts because they learn only from sparse scalar rewards at the end of each rollout. GEPA instead treats every execution trace as a rich, interpretable learning signal — the same kind a human engineer would use when debugging a system. One rollout's feedback can produce a large behavioral update.

Three Pillars

(Re)flection

An LLM reads the execution traces, tool outputs, and evaluation feedback (e.g. "compiler error: type mismatch on line 42"), diagnoses what went wrong, and proposes a revised prompt. The prompt update is natural language — interpretable, composable, and high-signal.

(Ge)netic

Each new candidate inherits the accumulated "lessons" from its parent in an evolutionary tree. Unlike one-shot prompt optimization, GEPA builds knowledge across generations — descendant prompts carry forward strategies discovered by their ancestors.

(Pa)reto Sampling

Instead of always mutating the single best candidate (which quickly stalls), GEPA maintains the top performer for each training instance, forming a Pareto frontier. It then stochastically samples from this frontier — balancing exploration of diverse strategies with exploitation of proven ones.

Paradigm Shift

Reinforcement Learning (GRPO)

  • × Learns from scalar rewards only
  • × Requires 24,000+ rollouts per task
  • × Needs weight updates (LoRA or full finetune)
  • × Cannot optimize closed-source models
  • × Aggregate gain: +3.68% (Qwen3 8B)

GEPA (Natural Language Reflection)

  • ✓ Learns from full execution traces
  • ✓ Optimal in 402–7,051 rollouts (up to 78x fewer)
  • ✓ Prompt-only — no weight updates needed
  • ✓ Works on GPT-4.1 Mini, Gemini, any API model
  • ✓ Aggregate gain: +9.62% (Qwen3 8B)

The Feedback Function

GEPA extends the standard evaluation metric into a feedback function that returns both a numeric score and natural language traces — compiler errors, rubric failures, profiling data. These traces are the diagnostic signal that makes reflection possible.

Core Algorithm

Formula 1: Per-Instance Pareto Frontier

Pareto Front Construction
$$s^*[i] = \max_{k} S_{\mathcal{P}[k]}[i]$$

For each training instance i, record the highest score achieved across all candidates in the pool. This defines the Pareto frontier — the set of "winning" strategies discovered so far.

Formula 2: Selection Probability

Weighted Sampling
$$P(\Phi_k) \propto f[\Phi_k] = |\{i : \Phi_k \in \hat{\mathcal{P}}^*[i]\}|$$

Candidates are sampled proportionally to how many instances they lead on. A candidate that wins on 5 tasks is 5x more likely to be selected than one that wins on 1 — but the "narrow specialist" still gets explored.

Formula 3: The Optimization Loop

Full GEPA Algorithm
$$\Phi' = \mathrm{Copy}(\Phi_k) \text{ with module } j \text{ updated by } \pi_j' \leftarrow \mathrm{Reflect}(\text{traces, feedback})$$

The LLM reflectively attributes successes or failures to prompt elements, proposes a revised instruction, and the new system is tested on the same minibatch. Only improvements are added to the candidate pool.

Optimization Flow

  1. Initialize: Candidate pool \(\mathcal{P} = [\Phi]\) with the base system. Split training data into feedback set and Pareto set.
  2. Pareto Selection: Build per-instance Pareto frontier from scores. Stochastically sample a candidate to mutate, weighted by how many instances it leads.
  3. Module Selection: Round-robin choose which module's prompt to update (ensures all modules get attention).
  4. Rollout & Reflect: Run the candidate on a minibatch. Collect execution traces, tool outputs, and evaluation feedback. An LLM reflects on these and proposes a new prompt.
  5. Minibatch Test: If the new prompt improves on the minibatch, add it to the candidate pool with ancestry records, then evaluate on the full Pareto set.
  6. Repeat until budget \(B\) is exhausted. Return the candidate with the best aggregate score on the Pareto set.

Case Study: Adversarial Prompt Search

GEPA's reward signal can be inverted — instead of maximizing performance, it minimizes it. The result: a universal adversarial instruction that works across all queries.

  1. Setup: Start from a clean instruction prompt for AIME math problems. Target: GPT-5 Mini, AIME-2025 (30 problems, 5 runs each = 150 generations).
  2. Evolution: GEPA iteratively proposes prompt edits that add information (e.g., trivia) while keeping the task solvable. Goal: reduce pass@1.
  3. Result: A single adversarial instruction reduced pass@1 from 76% to 10% — a 3.8x increase in error rate. The injected distractors (trivia about honey, the Nile, dolphins) caused systematic misinterpretation of the formatting rule.
  4. Why it matters: This shows GEPA's reflection is powerful enough to find brittle instruction-following interactions automatically — producing reusable stress tests for model safety evaluation.

Boundaries & Limitations

When It Stalls

If the Pareto frontier converges to a single strategy and all mutations fail to improve, GEPA exhausts its budget without finding better prompts. This happens when the search space is too narrow or the feedback function provides weak diagnostic signal.

Merge Timing

GEPA+Merge improves GPT-4.1 Mini but degrades Qwen3 8B on some tasks. The crossover (Merge) was invoked with the same hyperparameters for both models, which was suboptimal for Qwen. Merge works best only after sufficiently different lineages have evolved.

Validation Budget

The majority of GEPA's rollout budget is spent evaluating candidates on the Pareto (validation) set for selection, not for learning. Further efficiency gains are possible by using a smaller validation set or dynamic subsets.
